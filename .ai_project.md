# mlx-tts-benchmarks

## Purpose

Local benchmark archive for MLX-oriented TTS model comparisons, including generated audio, manifests, ASR scores, summaries, and reports.

## Repository Shape

- Top-level dated directories hold benchmark runs and comparison artifacts.
- Common artifact types: `wav`, `csv`, `json`, `md`, `txt`, `log`, `aiff`.
- The repository was initialized from an existing benchmark results folder, not from a source package.
- `index.html` is the GitHub Pages listening page for the latest published test audio.

## Conventions

- Keep benchmark run directories self-contained.
- Preserve manifests, summaries, reports, and scored outputs with their corresponding audio files.
- Do not delete or rewrite existing benchmark artifacts unless explicitly requested.

## Verification

- Project metadata inferred from the existing directory contents.
- No automated test suite is currently known.
- Static site verification should include checking that every `audio` source in `index.html` points to an existing file.
